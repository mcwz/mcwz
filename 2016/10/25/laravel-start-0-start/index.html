学习目标

该系列文章是为 Laravel 初学者准备的，建议有一定 PHP 基础且有打算学习 Laravel 框架的同学阅读。文章前半部分会介绍一下初学者要涉及到的概念，然后在后半部分会找一个小项目由浅入深逐步讲解，希望阅读前边章节的同学耐住寂寞，坚持按顺序阅读一下，我会尽可能的只涉及需要的概念，因此，该系列文章仅供入门学习用，如果有哪位同学赏光读到了最后，觉得文章还不错，想继续使用 Laravel 的，还是需要重新阅读官方文档，不过当你再次阅读时，应该就不是“天书”了。

使用 Laravel 版本说明

本教程使用的是 Laravel 5.3 ，区别于 5.1 LTS 这是一个一般版本，官方会提供六个月的 BUG 修复及一年的安全性修复。对于学习或者个人项目，建议使用 5.3 ，这样不仅能够经常更新且能够快速使用 Laravel 新特性。但是正因为更新频繁，这对于大型团队项目来说是个缺点，因为大型团队项目通常更追求“稳”。

Composer

以上说了一些不得不说的废话，下边将进入正题。

Composer 是 PHP 的一个依赖管理工具。有了它你就可以把现代化的框架看成一个个模块，看着哪个模块不爽，就可以去 Composer 里看看，如果有更好的，你只需要一行配置即可替换掉现有的模块。

那么首先我们要安装 Composer ， 访问官网 https://getcomposer.org/ 并下载安装。 安装完毕后在控制台（ windows 下的 cmd ，Linux 的 console ）下执行 Composer , 如果显示了版本号而不是提示找不到命令，那么恭喜你安装成功了。

  • PDO PHP Extension

  • redis

HomeStead

Homestead 是一个官方预封装的 Vagrant box ，使用它无需考虑系统、PHP、MySQL 等各种配置事项，可以把全部精力放到开发程序上。建议初学者使用该套环境，以保证不至于被各种软件配置分散大部分精力，不过不知道什么原因，在下载 Vagrant box 很慢很慢，我试了各种下载方式，提前下载了一份，放到了云上 链接: http://pan.baidu.com/s/1gfa8bar 密码: n9y6 ，请先下载。

HomeStead 已经预置了以下软件，并且已经配置完毕：

  • Ubuntu 16.04
  • Git
  • PHP 7.0
  • Nginx
  • MySQL
  • MariaDB
  • Sqlite3
  • Postgres
  • Composer
  • Node (With PM2, Bower, Grunt, and Gulp)
  • Redis
  • Memcached
  • Beanstalkd

安装步骤：

  1. 官网下载最新的 Vagrant （ http://www.vagrantup.com/downloads.html ），并安装。
  2. 下载最新的 VirtualBox （ https://www.virtualbox.org/wiki/Downloads ），并安装。
  3. 在控制台定位到之前下载的 Vagrant box 目录，然后执行 vagrant box add laravel/homestead virtualbox.box 。
  4. 执行 git clone https://github.com/laravel/homestead.git Homestead ，如果提示找不到命令，请先安装 git 。
  5. 在控制台中进入刚刚 clone 出的 HomeStead 目录,执行 bash init.sh 。
  6. 此时在用户路径下会生成 .homestead 目录，Windows 通常在 C:\Users***.homestead 。
  7. 打开该目录下的 Homestead.yaml ，我们现在仅关注两个配置，一个是 folders ，代表本机与虚拟机下路径共享配置，一个是 sites ，是网站虚拟主机的配置。
  8. 我们在宿主机上选一个工作目录，并把这个目录配置到 folders 下的 map 后边，当虚拟机启动后，这个目录将会共享给虚拟机使用，我们修改该目录下的代码相当于也修改了虚拟主机下的内容。
  9. 然后我们给 sites 下对应的 map 键设置一个自己喜欢的开发域名，如 mysite.com ，并在 hosts 下增加 192.168.10.10 mysite.com 记录。接着把 folder 下 to 的值拷贝到 sites 下的 to 后面，并增加一级目录 mysite 。这样以后在 mysite 下修改代码，使用 mysite.com 即可访问 ，不行等教程跑完，你可以在 mysite 下新建一个 index.php 试试。当然如果是 Laravel 的话，需要把 to 继续指到 mysite/public 下。
  10. 在控制台执行 vagrant up ，启动虚拟机。稍等片刻，如果没有报错。那么恭喜你的 HomeStead 配置成功啦。

开始安装 Laravel

  1. 在控制台执行 composer create-project –prefer-dist laravel/laravel project
  2. 如果不是 HomeStead ，请将虚拟主机根目录指到 public
  3. 将 storage 、bootstrap/cache 设置成可写模式
  4. 修改根目录 .env 配置。通常大部分配置信息是从这里读取的，如数据库连接信息、缓存服务器连接信息等。
  5. 为了程序安全，执行 php artisan key:generate 为程序生成一个新的随机字符串的密钥。
  6. 访问 http://localhost (或者是自己设置的域名)，如果看见了 Laravel ，说明 Laravel 安装成功了。

Laravel 目录结构

打开刚刚安装好的 laravel　根目录，以下将逐一将常见的目录介绍一下以便于对整个框架有个总体了解。

  • app\ 目录: 我们的大部分逻辑代码都要写到这里边，例如 Controllers 。
  • app\Http\ 目录：controller 、 中间件等存放目录。
  • app\Providers\ ：服务定义目录，系统提供的、自己写的服务都来这里定义。
  • config\ ：配置目录，包含系统、数据库、缓存、队列、session等配置，默认情况下，系统约定先去根目录 .env 里找对应配置，找不到的话会应用此处配置。随意打开一个配置文件，可以看到 env(‘‘, ) 语句，意思就是先去 .env 里读，读不到就用后边的默认值。
  • database\migrations\ ：数据库迁移（ migrate ）文件存放位置
  • public\ ：建议将此目录设置为虚拟主机根目录。这里边只有一个 index.php 是 php 文件，该文件是程序入口。通常情况下 css 、 JS 、图片等文件存放在 public 下。
  • resources\views\ ：视图文件存放位置
  • routes\ ：路由定义目录，其中 web.php 主要定义来自 web 请求的路由，也是该教程重点讲述的内容。
  • storage\ ：存一些日志文件、框架运行时文件、或者上传文件等。
  • tests\ ：写测试用例的目录。
  • vendor\ ： Composer 安装的模块全部在这里，也有 Composer 自动加载等逻辑代码。当执行 Composer require * ,Composer update 后，下载的模块以及依赖都在这里。
  • .env ：配置信息
  • composer.json ： Composer 配置信息。
